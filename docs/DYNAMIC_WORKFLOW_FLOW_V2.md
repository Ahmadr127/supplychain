## 3. WORKFLOW SELECTOR LOGIC

```php
/**
 * Workflow Selection berdasarkan:
 * 1. Jenis Pengadaan (BARANG_BARU / PEREMAJAAN)
 * 2. Nominal (Total Value)
 */
function selectWorkflow(string $procurementType, float $totalValue): ?ApprovalWorkflow
{
    // Determine nominal range
    if ($totalValue <= 10000000) {
        $nominalRange = 'low';      // â‰¤ 10 Juta
    } elseif ($totalValue <= 50000000) {
        $nominalRange = 'medium';   // 10 - 50 Juta
    } else {
        $nominalRange = 'high';     // > 50 Juta
    }
    
    return ApprovalWorkflow::where('is_active', true)
        ->where('procurement_type', $procurementType)  // BARANG_BARU atau PEREMAJAAN
        ->where('nominal_range', $nominalRange)
        ->orderByDesc('priority')
        ->first();
}

// Contoh penggunaan:
$workflow = selectWorkflow('BARANG_BARU', 75000000);
// â†’ Returns: "Pengadaan Barang Baru > 50 Juta"
```

## 4. CAPEX ID NUMBER MANAGEMENT

### Tabel: `capex_id_numbers`

Tabel terpisah untuk mengelola ID Number CapEx yang dipilih oleh Manager Unit saat approval.

```sql
CREATE TABLE capex_id_numbers (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    
    -- Capex identification
    capex_number VARCHAR(50) NOT NULL UNIQUE,      -- e.g., "CAPEX-2026-001"
    capex_year INT NOT NULL,                        -- Tahun anggaran
    capex_category VARCHAR(100),                    -- Kategori capex
    
    -- Budget information
    budget_amount DECIMAL(15,2) NOT NULL,           -- Total anggaran capex
    used_amount DECIMAL(15,2) DEFAULT 0,            -- Jumlah yang sudah terpakai
    remaining_amount DECIMAL(15,2) GENERATED ALWAYS AS (budget_amount - used_amount) STORED,
    
    -- Status
    status ENUM('active', 'exhausted', 'closed') DEFAULT 'active',
    
    -- Metadata
    description TEXT,
    department_id BIGINT REFERENCES departments(id),  -- Departemen pemilik
    created_by BIGINT REFERENCES users(id),
    approved_by BIGINT REFERENCES users(id),
    approved_at TIMESTAMP,
    
    -- Timestamps
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    -- Indexes
    INDEX idx_capex_year (capex_year),
    INDEX idx_capex_status (status),
    INDEX idx_capex_department (department_id)
);
```

### Tabel: `capex_allocations`

Tracking penggunaan CapEx ID Number untuk setiap approval request item.

```sql
CREATE TABLE capex_allocations (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    
    -- References
    capex_id_number_id BIGINT NOT NULL REFERENCES capex_id_numbers(id),
    approval_request_id BIGINT NOT NULL REFERENCES approval_requests(id),
    approval_request_item_id BIGINT REFERENCES approval_request_items(id),
    
    -- Allocation details
    allocated_amount DECIMAL(15,2) NOT NULL,        -- Jumlah yang dialokasikan
    allocation_date DATE NOT NULL,
    
    -- Status
    status ENUM('pending', 'confirmed', 'cancelled', 'released') DEFAULT 'pending',
    
    -- Approval tracking
    allocated_by BIGINT REFERENCES users(id),       -- Manager Unit yang memilih
    confirmed_by BIGINT REFERENCES users(id),       -- Yang mengkonfirmasi
    confirmed_at TIMESTAMP,
    
    -- Notes
    notes TEXT,
    
    -- Timestamps
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    -- Indexes
    INDEX idx_allocation_capex (capex_id_number_id),
    INDEX idx_allocation_request (approval_request_id),
    INDEX idx_allocation_status (status)
);
```
